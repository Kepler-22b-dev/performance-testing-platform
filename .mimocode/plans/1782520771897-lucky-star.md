# Plan: 为任务创建逻辑添加单元测试

## Context
项目中没有任何单元测试基础设施。我们需要为刚才修改的任务创建和清理逻辑创建基础测试，验证修改是否正确。

## 修改的文件回顾
1. `manager/core/scheduler.py` — `_cleanup_stuck_tasks`、`create_task`、`start_task`、`get_running_tasks`
2. `manager/api/tasks.py` — quick_run timeout 修复、`model_dump()` 替换
3. `manager/api/scheduler_api.py` — 新增 cleanup API

## 步骤

### 1. 安装 pytest
在 `manager/requirements.txt` 中添加 pytest

### 2. 创建测试目录和文件
- `tests/__init__.py`
- `tests/conftest.py` — mock 数据库和 Redis 的 fixture
- `tests/test_scheduler.py` — 测试 scheduler 核心逻辑
  - `test_create_task_empty_agents` — 空 agent 列表应抛 ValueError
  - `test_create_task_invalid_timeout` — timeout <= 0 应抛 ValueError
  - `test_cleanup_stuck_tasks_no_results` — 无结果的超时任务应被标记 failed
  - `test_cleanup_stuck_tasks_partial_results` — 部分结果的超时任务应被标记 failed
  - `test_cleanup_stuck_tasks_within_timeout` — 未超时的任务不应被清理
  - `test_start_task_missing_script` — 脚本不存在时 start_task 应返回 False 并标记 failed
- `tests/test_tasks_api.py` — 测试 API 层
  - `test_batch_create_uses_model_dump` — 验证 model_dump 调用正确

### 3. 运行测试
```bash
cd /Users/gungun/Downloads/QoderProject/PerformanceTestingPlatform
python -m pytest tests/ -v
```
