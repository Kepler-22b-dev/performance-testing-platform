# 性能测试平台 - 部署手册

> 版本：3.1 | 更新时间：2026-07-07

---

## 一、环境要求

| 组件 | 要求 | 说明 |
|------|------|------|
| Python | 3.10+ | 运行 Manager 和 Agent |
| Redis | 6.0+ | 消息队列和数据持久化 |
| Java | 8+ | JMeter 运行需要 |
| JMeter | 5.6.3 | 已内置在项目中 |
| 操作系统 | Linux / macOS / Windows | 推荐 Linux/macOS |

---

## 二、一键部署（推荐）

```bash
git clone https://github.com/Kepler-22b-dev/performance-testing-platform.git
cd performance-testing-platform
bash deploy.sh
```

脚本会自动：
1. ✅ 检测 Python、Redis、Java 环境
2. ✅ 自动安装缺失组件
3. ✅ 测试并选择最快的 pip 镜像源（阿里云/清华/腾讯等）
4. ✅ 自动安装所有 Python 依赖
5. ✅ 配置 JMeter
6. ✅ 启动所有服务

### 一键修复

```bash
bash deploy.sh fix
```

修复功能：检查依赖、切换镜像、启动 Redis、清理进程、重启服务。

## 四、手动部署

### 4.1 安装依赖

```bash
# 使用国内镜像（推荐）
pip3 install -r manager/requirements.txt -i https://mirrors.aliyun.com/pypi/simple/
pip3 install -r agent/requirements.txt -i https://mirrors.aliyun.com/pypi/simple/
```

### 4.2 配置 pip 镜像源

```bash
mkdir -p ~/.config/pip
cat > ~/.config/pip/pip.conf << EOF
[global]
index-url = https://mirrors.aliyun.com/pypi/simple/
trusted-host = mirrors.aliyun.com
timeout = 60
EOF
```

### 4.3 启动 Redis

```bash
# macOS
brew services start redis

# Ubuntu/Debian
sudo systemctl start redis && sudo systemctl enable redis

# 直接启动
redis-server --daemonize yes
```

### 4.4 启动服务

```bash
# 启动 Manager
nohup python3 -m manager.main > /tmp/manager.log 2>&1 &

# 启动 Agent
nohup python3 -m agent.main > /tmp/agent.log 2>&1 &

# 启动 JMeter Slave（可选）
bash start-slave.sh 1100
```

### 4.5 验证服务

```bash
curl http://localhost:8000/api/health
redis-cli ping
```

## 五、分布式部署

### 5.1 架构图

```
┌─────────────────────────────────────────────────────────┐
│                    管理节点                               │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │ FastAPI  │  │  Redis   │  │  Web UI  │              │
│  │  :8000   │  │  :6379   │  │  :8000   │              │
│  └──────────┘  └──────────┘  └──────────┘              │
└───────────────────────┬─────────────────────────────────┘
                        │ Redis PubSub
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
┌───────────────┐ ┌───────────────┐ ┌───────────────┐
│   Agent 1     │ │   Agent 2     │ │   Agent N     │
│   :9999       │ │   :9999       │ │   :9999       │
├───────────────┤ ├───────────────┤ ├───────────────┤
│ Slave :1100   │ │ Slave :1200   │ │ Slave :1300   │
└───────────────┘ └───────────────┘ └───────────────┘
```

### 5.2 管理节点部署

```bash
pip3 install -r manager/requirements.txt -i https://mirrors.aliyun.com/pypi/simple/

# 配置 Redis
sudo vim /etc/redis/redis.conf
# bind 0.0.0.0
# requirepass your_password

nohup python3 -m manager.main > manager.log 2>&1 &
```

### 5.3 施压节点部署

```bash
pip3 install -r agent/requirements.txt -i https://mirrors.aliyun.com/pypi/simple/

export REDIS_HOST=管理节点IP
nohup python3 -m agent.main > agent.log 2>&1 &

# 启动多个 Slave
bash start-slave.sh 1100 &
bash start-slave.sh 1200 &
```

### 5.4 节点注册

1. 登录 Web UI → 施压节点
2. 输入 Agent 的 IP 和端口
3. 点击「添加」→「验证」

### 5.5 防火墙配置

```bash
# Manager 节点
sudo firewall-cmd --add-port=8000/tcp --permanent
sudo firewall-cmd --add-port=6379/tcp --permanent

# Agent 节点
sudo firewall-cmd --add-port=9999/tcp --permanent
sudo firewall-cmd --add-port=1100-1301/tcp --permanent
sudo firewall-cmd --reload
```

## 六、硬件配置

### 配置方案

| 方案 | 适用场景 | 管理节点 | 施压节点 | 并发能力 |
|------|----------|----------|----------|----------|
| 开发测试 | 个人开发、功能测试 | 4核 8GB | 无（合一） | 200-500 |
| 小团队 | 5-20人团队 | 4核 8GB | 8核 16GB ×2 | 1000-2000 |
| 中大型团队 | 20-50人团队 | 8核 16GB | 8核 16GB ×3-5 | 2000-5000 |
| 企业级 | 50+人团队 | 16核 32GB ×2 | 16核 32GB ×5-10 | 5000-10000 |

### JVM 配置

```bash
# 编辑 apache-jmeter-5.6.3/bin/jmeter
JVM_ARGS="-Xms4g -Xmx6g -XX:+UseG1GC -XX:MaxGCPauseMillis=200"
```

| 并发数 | JVM 堆内存 | 服务器内存 |
|--------|-----------|-----------|
| < 200 | 2GB | 8GB |
| 200-500 | 4GB | 16GB |
| 500-1000 | 6GB | 16GB |
| 1000-2000 | 8GB | 32GB |

### Redis 配置

```conf
# /etc/redis/redis.conf
maxmemory 4gb
maxmemory-policy allkeys-lru
save 60 10000
tcp-backlog 511
```

## 九、日志查看

### 日志文件

| 日志文件 | 用途 | 记录内容 |
|----------|------|----------|
| `logs/manager.log` | Manager 服务日志 | 服务启动/关闭、Redis连接 |
| `logs/api.log` | API 请求日志 | HTTP 请求方法、路径、状态码、耗时 |
| `logs/agent.log` | Agent 运行日志 | Agent 启动/停止、心跳状态 |
| `logs/task.log` | 任务事件日志 | 任务生命周期事件 |
| `logs/error.log` | 错误日志 | 异常堆栈、错误详情 |

### 查看方法

```bash
# 实时跟踪日志
tail -f logs/manager.log

# 搜索特定任务
grep "task-abc123" logs/task.log

# 搜索错误
grep "ERROR" logs/error.log

# 统计 API 请求数
wc -l logs/api.log
```

### Web UI 查看

1. 打开 http://localhost:8000
2. 进入「工具日志」页面
3. 选择日志类型和行数
4. 点击「刷新」查看

## 十、故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| pip 安装超时 | 网络问题或源不可用 | `bash deploy.sh fix` 自动切换镜像 |
| Agent 无法连接 | Redis 未启动 | `redis-cli ping` 检查 |
| JMeter 执行失败 | Java 版本不兼容 | 安装 Java 8+ |
| iOS 设备无法检测 | pymobiledevice3 未安装 | `pip3 install pymobiledevice3` |
| Android 设备无法检测 | SOLOX 未启动 | `python3 -m solox` |
| 内存不足 | 并发线程数过高 | 增加内存或降低并发数 |

### 健康检查

```bash
curl http://localhost:8000/api/health    # API 健康检查
redis-cli ping                            # Redis 检查
curl http://localhost:8000/api/nodes/     # Agent 状态
curl http://localhost:8000/api/mobile/detect  # 移动设备检测
```

## 十一、安全建议

1. **Redis 安全**：配置密码、限制访问 IP
2. **网络安全**：使用防火墙限制端口访问
3. **数据安全**：定期备份脚本和报告
4. **访问控制**：生产环境建议添加认证机制
5. **日志管理**：定期清理过期日志和报告
