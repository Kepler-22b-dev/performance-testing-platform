# 性能测试平台演进路线图

本文档记录平台后续演进想法、落地优先级和维护备注，按业务需要逐步补齐。

## 平台定位

目标不只是执行 JMeter 脚本，而是逐步升级为**内部性能风险评估与容量决策平台**。

---

## 架构演进方向

### 插件化测试引擎

当前深度绑定 JMeter，建议设计插件式引擎层：

```
┌─────────────────────────────────────┐
│         统一任务调度层               │
├──────┬──────┬──────┬───────────────┤
│ JMeter│  k6  │ Locust│ 自定义引擎   │
│ 引擎  │ 引擎 │ 引擎  │  (SDK)       │
└──────┴──────┴──────┴───────────────┘
```

- 定义 `EnginePlugin` 抽象接口（`build_script` / `execute` / `parse_results`）
- 各引擎以独立模块注册，按需加载
- 不同场景选不同引擎：JMeter 适合协议级压测，k6 适合 JS 生态，Locust 适合 Python 自定义逻辑

### 事件驱动 + 可观测性中台

当前 Redis PubSub 是单向通知，建议升级为事件总线：

- 统一事件格式：`{event_type, task_id, timestamp, payload, metadata}`
- 支持事件订阅/过滤/重放（用于审计、回溯）
- 与外部可观测性系统（Prometheus/Grafana、OpenTelemetry）打通
- 压测任务自动关联被测服务的 Trace/Span，报告中直接定位瓶颈服务

### 时序数据存储

PostgreSQL 存业务元数据，时序数据（TPS/RT/错误率）迁移到 InfluxDB 或 TimescaleDB：

- 海量历史数据高效查询（按小时/天/周聚合）
- 自动降采样（7天明细 → 7天-30天小时级 → 30天+天级）
- 为容量建模、趋势预测提供数据基础

### Agent 弹性调度

当前 Agent 是静态部署，建议引入资源池化调度：

- Agent 注册时上报资源（CPU/内存/网络/磁盘IO）
- 任务调度根据资源需求 + 当前负载自动分配节点
- 支持容器化 Agent（K8s Job / Docker Compose 动态启停）
- 任务优先级队列，高优任务可抢占低优任务的 Agent

## 近期（1-2月）— 价值最高

| 功能 | 说明 | 状态 |
|------|------|------|
| 性能基线与回归检测 | 按脚本/接口/场景沉淀 P95、P99、错误率、吞吐、资源水位，后续自动判断是否退化 | ⬜ 待开始 |
| CI/CD 门禁集成 | CLI `check` 扩展为可配置规则（P95<200ms, 错误率<0.1%），失败阻断流水线 | ⬜ 待开始 |
| 多接口场景编排 | 支持按业务比例混合压测（登录30%+查询50%+下单20%） | ⬜ 待开始 |
| 容量评估报告 | 压测后自动输出：当前稳定 QPS、瓶颈资源、推荐实例数 | ⬜ 待开始 |
| 图片资源加载场景 | 接口返回资源 URL 后自动拉取，校准真实页面流量 | ⬜ 待开始 |

## 中期（3-6月）— 体系化能力

| 功能 | 说明 | 状态 |
|------|------|------|
| 压力机资源池 | 按标签（机房/机型/空闲状态）自动调度，支持容器化弹性伸缩 | ⬜ 待开始 |
| 任务预约与排队 | 指定时间执行 + 资源冲突检测 + 排队可视化 | ⬜ 待开始 |
| 混沌压测 | 压测中叠加故障注入（网络延迟/丢包/实例重启），测试系统韧性 | ⬜ 待开始 |
| 多环境联动 | 同一脚本自动在 dev→staging→prod 逐环境执行，对比各环境表现差异 | ⬜ 待开始 |
| 可观测性关联 | 压测任务与服务端指标、日志、Trace 关联，报告里直接提示疑似瓶颈位置 | ⬜ 待开始 |
| 多引擎支持 | 在 JMeter 基础上逐步支持 k6 等轻量脚本引擎 | ⬜ 待开始 |

## 远期（6月+）— 平台壁垒

| 功能 | 说明 | 状态 |
|------|------|------|
| AI 辅助诊断 | 结合压测结果+服务指标+日志+Trace，自动生成瓶颈分析和优化建议 | ⬜ 待开始 |
| 自动容量建模 | 基于历史数据推算服务容量曲线、安全水位、扩容时间窗口 | ⬜ 待开始 |
| 全链路压测 | 从网关→应用→中间件→DB 全链路覆盖，自动识别木桶短板 | ⬜ 待开始 |
| 性能知识库 | 沉淀历史压测经验（哪些优化有效、哪些场景易出问题），新人可检索学习 | ⬜ 待开始 |
