# 变更记录

## 2026-07-04

本次提交围绕性能测试平台的可用性、可扩展性和报告准确性做了一轮集中增强。

### 首页与页面体验

- 优化首页布局，压缩低价值介绍模块，优先展示最近任务、运行关注、优先操作和系统架构。
- 系统架构默认展开，减少首页留白。
- 统一运行关注和优先操作卡片尺寸，长文案使用省略号和悬停提示。
- 顶部导航固定显示并自动填满宽度，提升长页面滚动时的操作稳定性。
- 多个列表页面补充分页，减少数据量增长后的页面卡顿。
- 修正创建任务、脚本管理、CSV 预览等页面的控件对齐问题。

### 创建任务与压测配置

- 创建任务新增 JVM 堆内存、错误日志采样、结果格式、压力机独占等配置。
- JVM 堆内存支持按压测场景自动推荐，并提供计算说明。
- 任务超时时间改为按持续时间、预热时间、分布式模式和图片加载场景估算，避免默认值过长。
- 增加 JMeter 常用参数入口，默认使用 CSV JTL 结果文件，XML 作为调试选项。
- CSV 文件选择改为弹窗预览模式，支持分页查看数据。

### 动态调压

- 支持运行中动态增加或减少并发线程数。
- 动态加压会创建独立压力段，基础任务和动态压力段的进度会聚合展示。
- 动态压力段结果写入 `segments/` 目录，并纳入最终报告解析。
- 停止任务时会同步停止动态压力段，避免遗留 JMeter 进程。

### 报告与结果分析

- 任务编号从短随机 ID 改为日期加自增编号，例如 `task-20260704-001`，页面展示为 `20260704-001`。
- 结果接口支持展示编号和真实目录编号互相解析，修复单接口曲线因编号不一致加载失败的问题。
- 报告接口拆分为摘要、接口时序、样本分页，避免摘要接口返回全量样本。
- 单接口分析改为按接口标签单独拉取聚合时序数据。
- 动态图表整合为一个大图，统一展示 TPS、响应时间、错误率、线程数和网络流量。
- 测试报告卡片增加稳定期 QPS、最大 QPS、峰值网络流量和总流量。
- 请求样本、错误样本、详情查看支持分页读取。

### 日志能力

- 新增工具日志功能，用于排查平台、Agent、JMeter 执行过程问题。
- 错误响应日志改为限量采样和响应体截断，降低大流量场景下的磁盘与内存风险。
- 增加日志分析能力，展示 Top 问题、来源和详情。
- Top 问题长内容支持展开查看。
- 增加日志清理接口，支持按保留时间定期清理平台日志和任务日志。

### 数据结构与可扩展性

- 任务、脚本、CSV 列表支持后端分页和轻量字段查询。
- 样本缓存支持递归读取动态调压 `segments/` 目录。
- 补充 samples 聚合表设计备注，为后续按任务、接口、秒粒度落库预留方案。
- 增加结果保留策略能力，支持按天数、任务数和磁盘水位清理报告目录。

### Mock 电商压测场景

- 新增 mock 电商全流程接口：注册、登录、搜索商品、加入购物车、创建订单、查看订单详情、取消订单。
- 新增 10 分钟全流程 JMeter 脚本和用户 CSV。
- 支持分布式压测运行方式，mock 请求链路会写入 JSONL trace 作为留痕。
- 新增图片加载压测场景，用于模拟页面资源加载带来的真实流量。

### 文档与验证

- 新增 mock 电商压测说明文档。
- 新增 samples 聚合表设计备注文档。
- 补充动态调压、结果分页、编号解析、日志采样等相关测试。
- 本次回归验证通过完整 `pytest` 测试集。

### 本次提交不包含

- 本地工具计划文件 `.mimocode/`。
- 本地 Node 辅助文件 `package.json` 和 `package-lock.json`。
- 运行期生成的大体积 HTML 报告、缓存和日志文件。
